Project Title:
Automation framework for Web Application testing (konakart Enterprise Java eCommerce) Website.

URL : 
"https://www.konakart.com/konakart/Welcome.action".

----------------------------------------------------------------------------------------------------

Motivation:

Creating Automation Scripts To Test The Happy scenario , Test automation helps you reduce the feedback cycle and bring faster validation for phases in the development of your product. Test automation is especially useful because it helps you detect problems or bugs early on during the development phase, which increases the team's efficiency.

---------------------------------------------------------------------------------------------------

Technology/Framework Used:

Technology Used : Java, TestNG, Selenium, Maven.

Tools : Intellij IDEA Community Edition.

Framework Used: Page Object Model.

-----------------------------------------------------------------------

Features:

    • Smart Element Locators.
      
    • Conditional waiting.
      
    • Easy Assertions.
      
    • Reusable steps.
    • This project Automation Testing will help Tester to test his application using automation to reduce the manual time.
      
      
    • The Scenario in One test Case to Test The Fast of App and Measure the Response of application.

1- Install intellij IDEA Community Edition.
2- Install JDK and Check the Version .
3- Install Maven.
4- Install The Chrome Driver .
----------------------------------------------------------------------------------------------------------
Tests:

1- Open your The Automation tool.

The File Path (That you saved the Project).

How to use:

1- In Folder (Src) → tab Folder (Java) → Tab Folder (webPages) → (pageLocators) is contains all locators , Functions that used in MainClass .


2- In Folder (Src) → tab Folder (Java) → Tab Folder (Tests) → (MainClass. Java ) is contains The Scripts (Call Functions, orders) and annotations (@beforeMethod , @Test , @AfterMethod) that runs .

3- From Class (MainClass) Can run the test case and check the output .
